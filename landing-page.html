Visualizing Recursion

Look at stack frames, what they contain, and what they return to help trace recursive function calls.

Recursion can be one of the more difficult concepts to wrap your head around. This tool helps show what is happening when a recursive function is running and makes it easier to keep track of what's actually going on.

What this tool does

  • Step through a recursive function/algorithm one call at a time
  • See how stack frames are created, what they contain, and how they are removed
  • Track return values as the stack "unwinds"
  • Understand how recursion can scale and be useful in complex algorithms

How to Use It

  1. Choose an Algorithm
  2. Enter an input value
  3. Step through the recursive calls and returns

You get to control when the program moves on to the next step so you can see everything that is going on each time a call is made.

Having Trouble? Keep at it!

Recursion is difficult because you can't see what is actually happening each call until the stack finally unwinds. Compared to iterative/conditional loops print logging for each call is also much harder to grasp.
I can only hope that this tool is valuable enough to you so that it is less of a struggle to understand what is going on in each call of a recursive function.

Keep trying, it will get easier with more practice.
